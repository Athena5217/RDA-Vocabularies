4-fold path

The RDA Development Team is developing extensions to the RDA element sets to accommodate data from the "4-fold path".

There are three methods for using a string to identify a related entity:

  • An unstructured description: a plain literal, for example a free text note or a transcription.
  • A structured description: a literal typed with a form of syntax encoding scheme that aggregates literal values from other elements, for example a name/title access point.
  • An identifier: a literal that may be typed with a form of syntax encoding scheme that aggregates literal values from other elements, for example an ISBN or local identifier.

The fourth method uses a URI to identify the related entity as a linked data thing.

Datatype and object element sets

The RDA Vocabularies provide OWL datatype and object properties based on the RDA "canonical" properties. A datatype property can be used with or without a syntax encoding scheme for a value from one of the literal paths; an object property can be used for the URI.

Each datatype and object property is linked to its canonical property by an RDFS sub-property relationship.

The URI of each datatype and object property is derived from the URI of its canonical property by extending the namespace and retaining the most local part.

For example:

  • http://rdaregistry.info/Elements/w/P10198 (canonical)
  • http://rdaregistry.info/Elements/w/datatype/P10198 (datatype)
  • http://rdaregistry.info/Elements/w/object/P10198 (object)

Each datatype and object property has the same domain as its canonical property.

Each datatype and object property has the same label and lexical alias as its canonical property.

There are no definitions or scope notes for a datatype or object property. The human-readable semantics are associated with the canonical property.

Machine-readable semantics are expressed with object properties. The object property element sets contain, in particular:

  • representation of reciprocal RDA elements using the OWL inverse property.
  • representation of the target entity of RDA relationship elements using the RDFS range property.

Element set families

Canonical RDA properties are linked to associated unconstrained properties, that have no domains or ranges, by RDF sub-property relationships.

Most RDA elements are therefore represented in the RDA Vocabularies in a "family" of element sets.

The family consists of four member properties:

  • An unconstrained property with no declared domain or range.
  • A Toolkit canonical property with an RDA entity declared as the domain, but no range.
  • An object property with the same entity declared as the domain and an RDA entity declared as the range.
  • A datatype property with the same entity declared as the domain and a literal declared as the range.

Schema for the family of RDA element sets.

Example: "has creator"

domain URI domain property URI range URI range
unconstrained rdau:P60447
canonical rdac:C10001 work rdaw:P10065
datatype rdac:C10001 work rdawd:P10065
object rdac:C10001 work rdawo:P10065 rdac:C10001 agent

Class hierarchies

There are two hierarchies within the RDA Class element set.

They align RDA with the IFLA Library Reference Model (LRM).

Entity hierarchy

"RDA entity" is a super-class of all other RDA classes representing entities.

RDF graph of the class hierarchy of RDA Entity.

Agent hierarchy

"Agent" is a super-class of "Collective Agent" and "Person"; "Collective Agent" is a super-class of "Corporate Body" and "Family".

RDF graph of the class hierarchy of RDA Agent.

The hierarchies are also represented between properties using RDFS sub-property relationships.

Relationship matrix

A set of high-level, coarse granularity properties for relationships is distributed between the RDA element sets by domain.

The set covers all pair-wise associative relationships between the RDA entities.

Current and future RDA properties are treated as extensions of the matrix of relationships by using RDFS sub-property relationships.

RDF graph of a relationship matrix for the "has creator" element. All arcs are rdfs:subPropertyOf.

This example of a relationship matrix uses object, canonical, and unconstrained element set properties.

Properties labelled in italics are potential extensions to the element sets.

A similar matrix can be represented using the element's datatype properties.

Curie prefixes

Prefixes used in this documentation for a compact URI (curie):

prefix namespace element set
rdac: http://rdaregistry.info/Elements/c/ RDA Classes
rdau: http://rdaregistry.info/Elements/u/ Unconstrained properties
rdaw: http://rdaregistry.info/Elements/w/ Work properties
rdawd: http://rdaregistry.info/Elements/w/datatype/ Work datatype properties
rdawo: http://rdaregistry.info/Elements/w/object/ Work object properties
rdax: http://rdaregistry.info/Elements/x/ RDA Entity properties

A prefix for each canonical element set is suggested on the Registry webpage for the element set.
